## Suggested Improvements (Nice to Have)

### 1. Add client conversion rate to metrics dashboard

The dashboard tracks "active client subscriptions" but not the funnel: galleries delivered → clients invited → clients who pay. This ratio is the most important metric for determining if the business model works.

**Suggestion:** Add a row: "Client invite-to-pay rate (%)" — track how many invited clients actually subscribe.

### 2. Name specific Facebook groups before Phase 2

Phase 2 says "Join 1-2 Madison Facebook photography groups" but doesn't name them. Facebook groups often require applications and have strict rules.

**Suggestion:** Research and name 2-3 specific groups before Phase 2 begins. Add them to the strategy doc.

### 3. Desktop app is missing from outreach messaging

The desktop upload app handles large wedding galleries — a genuine differentiator vs. browser-only competitors. None of the referenced outreach templates mention it.

**Suggestion:** Add a note to Phase 1 onboarding: "Walk photographers through desktop app for large uploads. Mention in outreach when discussing wedding gallery delivery."

### 4. Add a "check-in if no gallery uploaded" protocol

The biggest Phase 1 risk: photographers sign up but never upload a gallery. This is the most likely failure mode.

**Suggestion:** Already partially addressed in Risk/Mitigation ("Follow up within 48 hrs"). Strengthen: "If a photographer hasn't uploaded a gallery within 7 days, send a personal check-in offering to walk them through it on a call."

### 5. Twig & Olive timing is appropriate in Phase 4

v1 critique suggested demoting Twig & Olive from Phase 3 to Phase 4. v2 correctly places them in Phase 4 ("need proof points first"). Good sequencing.

## What's Good

- **"The Core Insight" section is the most important paragraph in the plan.** "You only need to market to one side. Photographers bring their OWN clients." This prevents the common two-sided marketplace mistake of trying to build both sides simultaneously.

- **Phase gate criteria prevent premature advancement.** You can't enter Phase 4 without 20+ photographers, 5+ galleries, and 3+ paying clients. This prevents timeline pressure from overriding readiness.

- **"If You Can Only Do ONE Thing Per Phase" table is the plan's secret weapon.** On a bad week, Nate can look at one line and know what matters. This is the difference between a plan that gets executed and one that sits in a docs folder.

- **Email timing table prevents premature optimization.** "Build hero photo emails in Phase 4, not now" is correct. Too many solo founders build retention features before they have anyone to retain.

- **The ORB framework is applied cleanly.** Owned → Rented → Borrowed is the right priority order for a bootstrapped product with no audience.

- **Honest about what stalled and why.** "The gap was execution, not strategy" prevents rewriting a perfectly good strategy when the real problem was bandwidth.

## Summary

This is a clean, executable plan. The honest starting point (1 real photographer), realistic time budgets, and phase-gated progression make it trustworthy. The main risk is the same as v1: execution bandwidth. But the plan accounts for this with "minimum viable action" per phase and "priority override" when time is tight. Ready to execute.
